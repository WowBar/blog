## 基本检查

当数据类型操作不合理时，编译器静态编译时会提示错误。
```ts
let obj: { a: number; } = { a: 1 };
// A. 编译器推断出 obj 的类型: let obj: { a: number; }

// B. 编译器检查数据的使用是否合理，不合理时会抛出 Error
obj.b; // Error: 属性 b 不存在
obj.a = '2'; // Error：'2' 不能赋值给 number 类型
```
我们把以上编译器对类型的静态分析过程，分解成如下两个阶段：
+ A. 编译器推断出数据的类型：如注释 A。(第二节、第三节会对此深入探索)
+ B. 编译器检查类型的使用是否合理：如注释 B。(第四节[类型兼容][#类型兼容]会对此深入探索)

__解构__
```ts
// 数组解构
let [ foo = 'foo', ...bar ]: [number, ...string[]] = [ 1, 'bar' ];
// let [foo = 2, ...bar] = [1, 'bar'];
let foo: number
let bar: string[]
```

```ts
// 对象解构
let {
	foo = 2,
	bar: b,
	...baz
}: { foo: number, bar: string, baz: string } = {
	foo: 1,
	bar: 'bar',
	baz: 'baz'
};
// let { foo = 2, bar: b, ...baz } = { foo: 1, bar: 'bar', baz: 'baz' };
// let foo: number
// let b: string
// let baz: { baz: string; }
```

```ts
// 函数参数解构
function foo( { foo }: { foo: number } = { foo: 1 }, ...bar: any[] ): void { }
// function foo({ foo } = { foo: 1 }, ...bar) { }
```

## 类型推断
> Type Inference 类型推断可以保持代码的简洁性和可读性
+ _Basic Type Inference 基本类型推断_
+ _Best Common Type 最佳通用类型_
+ _Contextual Typing 上下文类型_

### 基本类型推断
在定义变量、函数时，编译器会进行如下类型推断：
```ts
// 在变量初始化时推断
let foo;
// let foo: any
let bar = 'ts';
// let bar: string

// 推断函数参数和返回值类型
let a = (x = 1) => {}
// let a: (x?: number) => void
```

### 最佳通用类型
从多个表达式的类型推断出最佳通用类型
```ts
let foo = ['ts', 1];
// let foo: (string | number)[]
```

### 上下文类型
由上下文推断出表达式的类型
```ts
let add: (a: number, b: number) => number;

add = function(x, y) {
	// 由 add 可推断出 x、y 以及返回值的类型
	return x + y;
};
// add = function(x: number, y: number): number {
// 	return x + y;
// };
```

## 类型断言
> Type Assertion 类型断言两种语法：
+ 尖括号语法: `<T>value`
+ as 语法: `value as T` (jsx 中只能用 as 语法)
```ts
let foo: any = 'this is a string';

(<string>someValue).length
// or
(someValue as string).length;
```

## 类型兼容
> Type Compatibility 类型兼容性 (TS 基于结构子类型设计，因为 JS 中广泛的使用匿名对象，例如函数表达式和对象字面量)

源类型 Y 可以赋值给目标类型 X，即 X = Y，此时我们说，目标类型 X 兼容源类型 Y。
```ts
interface TypeA {
	a: string;
}
interface TypeB {
	a: string;
}

let foo: TypeA = {a: 'ts'}
// 基于结构类型，所以 foo 可赋值给 bar，即 TypeB 兼容 TypeA
let bar: TypeB = foo;
```

### 基本类型兼容
之前“内置类型”篇幅中已有介绍，详情请[点击此处][built-in] 。
```ts
// `string` 兼容 `null`; `null` 是 `string` 的子类型
let str: string = null;
```

### 成员结构兼容
> 成员少的兼容成员多的

#### 基本结构兼容
源类型至少具有与目标类型相同的成员
```ts
let foo: { a: string } = { a: 'a' };
let bar: { a: string, b: string } = { a: 'a', b: 'b' };

foo = bar; // OK, 少兼容多
bar = foo; // Error
```
```ts
// 函数返回值
let foo = () => ({ a: 'a' });
let bar = () => ({ a: 'a', b: 'b' });

foo = bar; // Ok， 少兼容多
bar = foo; // Error
```

#### 类兼容
类的实例成员少的兼容成员多的，父类与子类的实例相互兼容
```ts
class Animal {
	feet: number;
	constructor(name: string, numFeet: number) {
		this.feet = numFeet;
	}
}
class Size {
	feet: number;
	constructor(numFeet: number) {
		this.feet = numFeet;
	}
}

let a: Animal = new Animal('cat', 1);
let s: Size = new Size(2);

a = s; // OK
s = a; // OK
```

#### 枚举兼容
枚举类型与数值类型相互兼容，枚举之间不兼容
```ts
enum Color { Red, Green }
enum E { X, Y }

// 枚举类型与数值类型相互兼容
let foo: Color = 123; // OK
let bar: number = Color.Red; // OK
// 枚举之间不兼容
let baz = Color.Red;
baz = E.X; // Error
```

#### 泛型兼容
```ts
interface Empty<T> {}
let x: Empty<number> = {};
let y: Empty<string> = {};

x = y; // OK
y = x; // OK
```
```ts
interface NotEmpty<T> {
	data: T;
}
let x: NotEmpty<number> = { data: 1 };
let y: NotEmpty<string> = { data: '1' };

x = y; // Error
```

### 函数参数个数兼容
> 参数多的兼容参数少的
```ts
// 参数个数
let foo = (a: number) => a;
let bar = (a: number, b: number) => a + b;

foo = bar; // Error
bar = foo; // OK
```
